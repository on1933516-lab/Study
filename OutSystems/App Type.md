# ■ OutSystems アプリタイプについて

## ■ OutSystems アプリタイプ

 - 従来の Web
 - モバイル（スマホ・タブレット）※
 - リアクティブ Web ※

 ※後段にて説明

## ■ 共通の特徴（モバイル / リアクティブ Web）

最新の規格とベストプラクティスに沿ったアーキテクチャで、リッチな UI/UX を提供

OutSystems がコードを自動コンパイルし、
ReactJS ベースの SPA（Single Page Application） を生成

ローコードでクロスデバイス向けのアプリを構築でき、開発スピードを大きく短縮

データは必要に応じて非同期で取得し、パフォーマンスを最適化

## ■ モバイル / リアクティブ Web それぞれの特徴比較表

| 項目 | **モバイルアプリ（Mobile App）** | **リアクティブ Web（Reactive Web）** |
|------|----------------------------------|---------------------------------------|
| **アクセス方法** | ・ネイティブアプリとしてデバイス上で実行<br>・または PWA としてブラウザで実行 | ・ブラウザ経由でアクセス（URLで利用） |
| **UI/UX** | ・小型デバイス向けに最適化された UI/UX が必要 | ・レスポンシブレイアウトで、画面サイズごとに自動適応 |
| **ネイティブ機能** | タッチ ID、カメラ、プッシュ通知など利用可能 | HTML5 が提供する範囲のみ |
| **オフライン対応** | あり（内部ストレージでデータ保持） | なし |
| **データ取得** | 非同期取得（SPAの仕組みは共通） | 非同期取得（SPAの仕組みは共通） |
| **アプリ更新** | ・ネイティブアプリ：自動更新可能<br>・PWA：新バージョン検出で自動更新 | ユーザーがブラウザをリフレッシュすると更新 |
| **配布方法** | ・アプリリンクの共有<br>・必要に応じてアプリストア | ・社内配布 / アプリストア<br>・PWA は Web から直接実行＋ホーム画面追加可 |
| **ハードウェア要件** | デバイスのネイティブ機能を幅広く活用可能 | HTML5 で利用できる範囲のみ |
| **ストレージ** | 内部ストレージ利用（オフライン可） | ブラウザのキャッシュ / LocalStorage のみ |

## ■ モジュラープログラミング
● モジュールとは

機能を実行するために必要な要素をひとまとめにした「カプセル化された単位」。

● モジュールの種類

 - **アプリタイプモジュール**：UI構築用（Reactive Web / Mobile App）

 - **ブランクモジュール**：空の状態から開始したい場合に使用

 - **サービスモジュール**：サービスロジックやドメインロジック（SOA）向け

 - **ライブラリモジュール**：再利用性の高いコンポーネントを提供

 - **エクステンションモジュール**：外部データやカスタムコードと連携する

 - **プロデューサモジュール**：機能や要素を公開する “提供側”

 - **コンシューマモジュール**：公開された機能を利用する “利用側”
