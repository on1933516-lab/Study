# UI 開発について
## ■ 1. 画面（Screen）

画面は UI の構成要素（ウィジェット）を組み合わせて作る単位。
以下の要素を組み合わせて、リッチな UI を構築できる。

 - リンク（Link）
 - ボタン（Button）
 - 画像（Image）
 - テーブル（Table / List）

## ■ 2. ウィジェット（Widget）

画面を構成する部品（UI 要素） の総称。

例：

 - タイトル（Text）
 - ボタン（Button）
 - リスト表示用テーブル（Table / List）
 - チェックボックス（Checkbox）
 - 画像（Image）
 - 入力フィールド（Input, Dropdown, TextArea など）

## ■ 3. レスポンシブデザイン

OutSystems の Web（Reactive）では、
デバイスの画面サイズに応じてレイアウトが自動調整される

 - PC
 - タブレット
 - スマホ

など、画面幅に応じて行や列が変化する。

## ■ 4. 画面の入力（Input Parameter）とローカル変数（Local Variable）
● 入力パラメータ

画面に値を渡すことができる変数。
例：詳細画面に「ID」を渡すなど。

● ローカル変数

画面内部のみで使用する変数。

画面のスコープ内で初期化

UI の一時的な状態管理に使用
（例：チェックボックス値、入力欄の状態など）

## ■ 5. ボタン・リンクの動作（イベント処理）
● 例：”Save” ボタン押下

ボタンの OnClick をトリガー として

画面アクション（Screen Action）やクライアントアクションが実行され

データベース更新などの処理が行われる

● リンクとボタンの違い

どちらも OnClick があり遷移や処理は可能だが、

リンク：よりナビゲーション寄りのUI

ボタン：操作アクション（更新・登録）向け

という UI 体験の違いがある。

## ■ 6. 画面アクション（Screen Action）

画面レベルで定義されるロジック

画面内のボタンやリンクなどがトリガー

クライアント側で実行される（Reactiveの場合）

## ■ 7. クライアントアクション（Client Action）

ボタン/リンクから呼び出されるロジック

モデル化されたフロー（Assign、If、Fetch、Ajax Refreshなど）

サーバーアクションの呼び出しも簡単に可能

● UIとの連動

UI でデータが変われば即座に反映

非同期更新（Ajax Refresh）も可能
