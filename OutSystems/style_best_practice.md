# CSS ベストプラクティス（OutSystems 向け）

## OutSystems に直接関連するベストプラクティス

### ■ クラスを使用・再利用する  
- ID、要素、ユニバーサルセレクタの使用は避ける  
- クラスの前に要素セレクタを付けない  
  例：  
  -  NG ⇒  `button.btn-primary`  
   - OK ⇒  `.btn-primary`

---

### ■ インラインスタイルは慎重に使用する  
強制度が非常に高く、テーマや外部CSSによる上書きが難しくなるため、**極力避ける**

---

### ■ コメントを使う／ファイル構造を明確にする  
- テーマファイルの先頭に目次を入れる  
- コード内で似た要素をグループ化する  

---

### ■ 命名規則  
- クラス名は**小文字＋単語間はハイフン (`-`)**  
- スタイル宣言は**アルファベット順**で整理  

```css
.btn-primary {
  background-color: #000;
  border-radius: 4px;
  color: #fff;
  padding: 8px;
}
```

---

### ■ プロパティと値の間にスペースを入れる

  例：  
   - NG ⇒  `color:#000;` 
   - OK ⇒  `color: #000;`

---

### ■ !important は使用しない

依存関係を壊し、後の保守が難しくなるため

---

### ■ グローバルな変更は CSS変数で行う

色・サイズ・フォントなどのテーマ全体に関わる値は変数で管理

---

### ■ レイアウトには Flexbox を使う

デバイスに応じたサイズ変更が可能。OutSystems UI でも標準的

---

### ■ テーマファイルの目次構成例

```text
テーマ名
$0 - インポートと変数
$1 - CSSリセット
      ブラウザのデフォルトスタイルをリセット
$2 - HTML要素の基本スタイル
$3 - ページレイアウトとグリッドシステム
$4 - パターン
$5 - ウィジェット（コンポーネントごとに分割）
$6 - スタイル（タイポグラフィ、色など）
```
