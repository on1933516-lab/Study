## CSSのベストプラクティス（OutSystems向け）

### OutSystemsに直接関連するベストプラクティス

■ クラスを使用・再利用する  
ID、要素、ユニバーサルセレクタの使用は避ける。  
また、クラスの前に要素セレクタを置かない。

例：  
NG ⇒  `button.btn-primary`  
OK ⇒  `.btn-primary`

---

■ インラインスタイルは慎重に使用する  
強制度が非常に高く、テーマや外部CSSによるスタイル上書きが難しくなるため。

---

■ コメントを使用すること  
・テーマファイルの先頭に目次を作成する  
・コード内で似た要素をグループ化し、管理しやすくする  

---

■ 命名規則  
・小文字を使用し、単語は「-」で区切る  
・スタイル宣言はアルファベット順に並べる  

例：  
```css
.btn-primary {
  background-color: #000;
  border-radius: 4px;
  color: #fff;
  padding: 8px;
}
```

---

■ プロパティと値の間にスペースを入れる  
（読みやすさ向上のため）  

NG ⇒  `color:#000;`  
OK ⇒  `color: #000;`

---

■ 「!important」は使用しない  
依存関係を壊し、後の修正が難しくなるため原則禁止。

---

■ グローバルな変更は CSS変数 を編集する  
色・サイズ・フォントなど全体に影響する部分は変数で管理することで、修正が一箇所で済み保守性が高くなる。

---

■ パターン利用時は Flexbox を使用する  
デバイスに応じて自動でサイズ変更が行える。  
（OutSystems UI でも Flexbox が基本構造として使用されている）

---

■ テーマの目次を作成する  

テーマファイルの構造例：  
```
テーマ名
$0 - インポートと変数
$1 - CSSリセット
      ブラウザのデフォルトスタイルをリセット
$2 - HTML要素の基本スタイル
$3 - ページレイアウトとグリッドシステム
$4 - パターン
$5 - ウィジェット
      コンポーネントごとに分割
$6 - スタイル（タイポグラフィ、色など）
```
