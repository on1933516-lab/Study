# S3について

## S3の概要

Amazon S3 とは、データをオブジェクト、階層データ、または表形式データとしてバケット内に保存するオブジェクトストレージサービスのこと。

複数のAZに保存され可用性を保ち、99.999999999％の高い耐久性（データロスが発生しない度合い）を実現。

特定のAZで障害が起きてもデータ消失を防げる。

オブジェクトとは、ファイル（画像ファイル・動画・PDF・JSON など）と、そのファイルを記述している任意のメタデータ（「作成者」「作成日」「ファイルタイプ」）のこと。

バケット とは、オブジェクトのコンテナのこと。

## ストレージ管理

 - S3 ライフサイクル：オブジェクトを他の S3 ストレージクラスに移行したり、ライフタイムが終了したオブジェクトを期限切れにできる。
 - S3 オブジェクトロック：Amazon S3 オブジェクトが固定期間または無期限に削除または上書きされるのを防止する。

## 主要機能

 - 静的Webサイトホスティング：
   バケットにWebサーバーの機能を付与し、HTMLファイルなどでWebサイトを作成できる。

 - アクセス制御：
   オブジェクトへのアクセスを制限するのに、ACL（アクセス許可の被付与者と付与されるアクセス許可を識別するリスト。基本的な読み取り/書き込み許可を他の AWS アカウントに付与する。）,バケットポリシー、パブリックアクセスブロック設定を提供している。

## ストレージクラス

| ストレージクラス              | 主な用途                          | アクセス頻度                   | 特徴・備考                                                         |
|------------------------------|-----------------------------------|------------------------------|--------------------------------------------------------------------|
| S3 Standard                  | 頻繁にアクセスされるデータ         | 高                         | 高パフォーマンス・複数AZ対応                                        |
| S3 One Zone-IA              | 長期間保管だがアクセス少ないデータ | 低                         | 単一AZ、コスト最適化用途                                            |
| S3 Standard-IA              | 長期間保存・アクセス少なめ          | 低                         | 複数AZ、IA用途向け                                                  |
| S3 Intelligent-Tiering       | アクセスパターンが不明／変化する   | 変化あり                    | 自動でストレージクラス切替、手間を省ける                             |
| S3 Glacier Instant Retrieval| アーカイブだが即時アクセスが必要   | 非常に低／即時アクセスあり  | 保存コスト低め、取り出し速い                                          |
| S3 Glacier Flexible Retrieval| 大量保存・取り出しに猶予がある     | 非常に低／即時不要          | 保存コストさらに低、リトリーブに時間あり                            |
| S3 Glacier Deep Archive      | 最も長期保管／アクセス不要         | 極めて低                   | 業界最安クラス、数時間〜数日でアクセス                                |
