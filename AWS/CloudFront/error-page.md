# エラーページ（Custom Error Response）

CloudFront が **オリジンから HTTP エラーコードを受け取ったときの挙動**をカスタマイズできる。

## 設定できる項目

### 1. レスポンスページの差し替え
任意の HTML（カスタムエラーページ）を返すことができる。  
例：403/404 を custom-error.html に置き換えて返す。

### 2. 最小 TTL（エラーのキャッシュ時間）
CloudFront が「このエラー結果を〇秒キャッシュしてよい」と判断する時間。

- **短くする** → 最新状態が反映されやすい  
- **長くする** → エラーを返す回数が減る（効率重視）

### 3. HTTP レスポンスコードの変更
返すステータスコードを変更できる。

- 例：403 → 200 にして「見た目上成功扱い」にする  
- SEO や UX の都合で使用されることがある
